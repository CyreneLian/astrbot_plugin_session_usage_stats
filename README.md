# 📊 astrbot_plugin_session_usage_stats

> 统计 AstrBot 全部模型在过去 24 小时 / 7 天 / 30 天内的调用次数与 Token 消耗，并提供精美的**模型用量统计仪表盘**、趋势排行、平台 / 会话辅助视图和低开销增量扫描。

## 🌸 模型用量统计插件

`astrbot_plugin_session_usage_stats` 是一个面向“模型用量统计”的 AstrBot 插件，用于统计全部模型在过去24小时/7天/30天内的调用次数、Token 消耗与趋势变化，并提供美观的网页仪表盘查看模型占比、平台占比、Top 会话、各会话对话轮数、趋势图和明细表格。

## ✨ 功能特性

- 支持网页仪表盘
  - 总览卡片
  - Top 会话排行
  - 平台占比圆环图
  - 模型占比圆环图
  - 趋势概览折线图
  - 明细表格搜索与排序
- 统计模型用量数据
  - 过去 24 小时
  - 过去 7 天
  - 过去 30 天
- 保留会话维度辅助统计，方便定位模型消耗来源
- 统计字段包括
  - 模型调用次数
  - 输入 Token
  - 输出 Token
  - 总 Token
  - 完成回复轮数
  - 用户消息数 / Bot 消息数
- 支持模型调用统计
  - 对话模型
  - 全部模型
  - Token / 次数切换
- 支持 QQ / aiocqhttp 等不稳定写入历史库的平台通过事件捕获兜底统计
- 支持清空统计数据，并通过 `clear_at` 避免旧历史数据重新回流

## 🧪 已测试平台

> ⚠️ **本插件目前仅在以下两个平台经过完整测试，统计结果可以保证准确。其它平台虽可启用，但尚未经过完整测试，统计可能不准确、不完整或完全无数据。**

| 平台 | 是否经过完整测试 | 说明 |
| --- | --- | --- |
| `webchat`（AstrBot 网页聊天） | ✅ 已测试 | 统计准确，默认即启用 |
| ↳ 含 OpenAPI（外部 HTTP / WebSocket API 调用） | ✅ 已涵盖 | 复用 webchat 的会话、umo 与队列（`platform_id="webchat"`，umo 前缀 `webchat:FriendMessage:webchat!...`），归入 webchat 平台统计 |
| `aiocqhttp`（QQ / OneBot） | ✅ 已测试 | 需开启 `enable_event_capture: true` 才能稳定统计 |
| 其它平台（wecom_ai_bot / telegram / discord / matrix / feishu 等） | ❌ 未测试 | 自行验证后再使用，请勿依赖此插件做关键统计 |

### 📌 其它平台使用建议

如果需要在未测试的平台上使用本插件，建议：

1. **先在测试场景验证核心数据**：消息轮数、Token 消耗、Top 会话等是否正常落库
2. **开启自动扫描**：`enable_auto_scan: true`，并适当调大 `scan_batch_size` 防止漏扫
3. **留意主历史库写入情况**：插件依赖 `platform_message_history` 表的历史数据，未测试平台可能写入不完整
4. **使用「会话统计诊断」命令**：`会话统计诊断` 可以查看 Provider wrapper 接管情况
5. **若该平台依赖主历史库外的数据源**：可能需要自行扩展统计逻辑

### ⚠️ QQ / aiocqhttp 必开配置

由于 QQ 平台的消息历史写入不够稳定，强烈建议在配置中启用事件捕获：

```json
{
  "enable_event_capture": true,
  "event_capture_platforms": ["aiocqhttp"]
}
```

否则 QQ 的统计可能出现"用户消息数 > Bot 消息数"或部分会话数据缺失。

## 📦 安装方式

将插件目录放入 AstrBot 插件目录：

```text
AstrBot/data/plugins/astrbot_plugin_session_usage_stats
```

目录结构通常如下：

```text
astrbot_plugin_session_usage_stats/
├── main.py
├── metadata.yaml
├── _conf_schema.json
├── logo.png
└── pages/
    └── stats/
        └── index.html
```

安装或更新后，请在 AstrBot 中重载插件，或直接重启 AstrBot。

## ⚙️ 配置项说明

配置项位于 `_conf_schema.json`，可在 AstrBot 插件配置界面中调整。

### enable_auto_scan

是否开启自动扫描。

- 类型：bool
- 默认值：true
- 说明：开启后，插件会按设定间隔自动增量扫描历史消息。关闭时，仅在查询统计或打开页面时进行补扫。

### auto_scan_interval_minutes

自动扫描间隔，单位为分钟。

- 类型：int
- 默认值：5
- 说明：仅在自动扫描开启时生效。

### scan_batch_size

单次扫描最大消息数。

- 类型：int
- 默认值：500
- 说明：用于限制一次扫描处理的历史消息数量，避免单次扫描过重。

### enabled_platforms

启用历史扫描的平台 ID。

- 类型：list
- 默认值：`["webchat", "aiocqhttp"]`
- 说明：只有在列表中的平台会被纳入历史消息扫描。与 `event_capture_platforms` **无需重复填写**，前者对应“历史扫描”数据源，后者对应“实时事件捕获”数据源，插件会自动处理好它们的数据合并。

### include_threads

是否统计 `webchat_thread`。

- 类型：bool
- 默认值：false
- 说明：开启后，如果 `enabled_platforms` 中没有 `webchat_thread`，插件会自动附加该平台。

### enable_event_capture

是否启用事件钩子捕获。

- 类型：bool
- 默认值：true
- 说明：适用于 QQ / aiocqhttp 等可能不稳定写入主历史库的平台。开启后，插件会在 Bot 完成回复时实时写入统计库。

### event_capture_platforms

事件钩子捕获的平台列表。

- 类型：list
- 默认值：`["aiocqhttp"]`
- 说明：仅在 `enable_event_capture` 开启时生效。与 `enabled_platforms` **无需重复填写**，前者对应“实时事件捕获”数据源，后者对应“历史扫描”数据源，插件会自动处理好它们的数据合并。

### auto_cleanup_enabled

是否开启自动清理旧统计数据。

- 类型：bool
- 默认值：true
- 说明：开启后会在插件启动时以及每次自动扫描后，自动清理超过保留天数的统计数据。

### auto_cleanup_retention_days

统计数据保留天数。

- 类型：int
- 默认值：30
- 说明：仅影响插件自身数据库中的统计数据，超过这个天数的数据会被自动清理。

pages/stats/index.html

仪表盘包含以下区域。

### ⏳ 时间范围切换

页面顶部支持切换：

- 过去 24 小时
- 过去 7 天
- 过去 30 天

### 📋 总览卡片

展示当前时间范围内的汇总数据：

- 会话数
- 总轮数
- 总 Token
- 输入 Token
- 输出 Token

### 🏆 Top 会话

展示当前周期内用量最高的前 10 个会话。

支持切换排序指标：

- Token
- 轮数

### 🥧 平台占比

以圆环图展示各平台占比。

支持切换指标：

- Token
- 轮数

### 🍩 模型占比

展示模型调用统计。

支持切换统计范围：

- 对话模型
- 全部模型

支持切换指标：

- Token
- 次数

说明：

- Token 占比会隐藏 0 Token 的模型调用。
- 嵌入、重排等没有 Token 信息的调用，通常只适合在“次数”维度查看。
- 模型数量较多时，前 8 个模型单独展示，其余合并到“其它”。
- 鼠标悬停“其它”项可查看被折叠的模型明细。

### 📈 趋势概览

展示当前周期内的趋势变化。

- 过去 24 小时：按小时展示
- 过去 7 天：按天展示
- 过去 30 天：按天展示

支持切换指标：

- Token
- 轮数

### 📑 明细表格

展示会话明细数据。

包含字段：

- 平台
- 会话 ID
- 轮数
- 用户消息
- Bot 消息
- 输入 Token
- 输出 Token
- 总 Token

支持：

- 搜索平台或会话 ID
- 按 Token 排序
- 按轮数排序
- 再次点击当前排序项切换升序 / 降序
- 最多显示 10 条，表格内部滚动查看更多

### 🔄 刷新数据

点击页面右上角“刷新”按钮，可以重新拉取当前周期数据。

### 🗑️ 清空数据

点击“清空数据”按钮后，需要在短时间内再次点击确认。

清空会删除：

- 会话统计数据
- 模型统计数据
- 模型调用缓存统计

清空后插件会对自身统计库执行整理，并重新准备空白统计状态；自动清理旧数据则只会删除超过保留天数的统计数据，不会执行磁盘整理。

## 📏 统计口径说明

### 🔄 轮数

轮数以 Bot 完成一次回复为锚点。

也就是说，一轮通常表示：

```text
用户消息触发了一次完整 Bot 回复
```

这样可以避免只统计到用户消息、但 Bot 没有实际完成回复时造成数据虚高。

### 🗣️ 用户消息数与 Bot 消息数

用户消息数和 Bot 消息数不一定永远相等。

可能出现差异的情况：

- 用户连续发送多条消息
- 用户中途打断
- 消息没有触发 Bot 回复
- 平台历史记录写入不完整
- 旧数据来自不同统计口径

当前插件会尽量以“完成回复”为核心口径，让统计更接近实际消耗。

### 🪙 Token

Token 分为：

- 输入 Token：Prompt / 上下文消耗
- 输出 Token：模型回复消耗
- 总 Token：输入 Token + 输出 Token

如果某些 Provider 或模型调用没有返回 Token 信息，相关 Token 可能为 0，但调用次数仍可能被记录。

### 🪟 滚动窗口

插件当前使用滚动窗口统计：

- 今日 = 过去 24 小时
- 本周 = 过去 7 天
- 本月 = 过去 30 天

不是自然日 0 点到 24 点，也不是自然周或自然月。

## 💾 数据存储

插件数据存储在插件数据目录下的 SQLite 数据库中：

```text
usage_stats.db
```

主要数据表包括：

- `usage_stats`：会话维度统计
- `model_usage_stats`：模型维度统计
- `model_call_stats`：模型调用统计缓存 / 兼容数据
- `scan_state`：扫描游标与清空时间

通常不建议手动修改数据库。如果需要清空数据，请优先使用网页仪表盘中的“清空数据”。

## 💡 推荐配置

### 只统计 WebChat

适合只使用 WebChat 的场景。

```json
{
  "enabled_platforms": ["webchat"],
  "include_threads": false,
  "enable_event_capture": false
}
```

### 统计 WebChat + QQ

适合同时使用 WebChat 和 aiocqhttp 的场景。

```json
{
  "enabled_platforms": ["webchat", "aiocqhttp"],
  "enable_event_capture": true,
  "event_capture_platforms": ["aiocqhttp"]
}
```

### 开启自动扫描

适合希望页面打开时更快看到数据的场景。

```json
{
  "enable_auto_scan": true,
  "auto_scan_interval_minutes": 5,
  "scan_batch_size": 500
}
```

如果消息量很大，可以适当调大扫描间隔，或降低单次扫描上限。

## ❓ 常见问题

### 1. 页面没有新数据

可以尝试：

```text
会话统计 补扫
```

或点击网页仪表盘右上角“刷新”。

如果刚刚修改过插件代码或配置，请重载插件或重启 AstrBot。

### 2. QQ 数据没有统计出来

请检查：

- `enabled_platforms` 是否包含 `aiocqhttp`
- `enable_event_capture` 是否开启
- `event_capture_platforms` 是否包含 `aiocqhttp`
- 插件是否已经重载

QQ / aiocqhttp 等平台可能不稳定写入主历史库，建议开启事件捕获兜底。

### 3. 用户消息数比 Bot 消息数多很多

可能原因：

- 用户连续发言
- 群聊中普通消息没有触发 Bot 回复
- 历史旧数据口径不同
- 事件捕获或历史扫描配置不一致

当前版本不会在收到用户消息时立即入库，而是在 Bot 完成回复后统一记录，以减少虚高。

### 4. 清空后旧数据又出现

当前版本已经使用 `clear_at` 机制避免清空后旧历史数据回流。

如果仍出现异常，请确认：

- 插件是否重载到最新版本
- 是否存在多个插件副本
- 是否手动恢复过旧数据库

### 5. 模型名称显示 unknown

可能原因：

- Provider 没有暴露模型名称
- 旧数据产生时还没有模型提取逻辑
- 某些调用不是标准对话模型调用

建议清空旧数据后，让新回复重新产生统计。

### 6. 模型 Token 为 0

部分调用可能没有 Token 信息，例如：

- **嵌入 (Embedding)**：底层 API 接口（如 OpenAI 的 `/v1/embeddings`）在 AstrBot 适配器中设计为仅返回特征向量列表（`list[float]`），其内部没有将 API 响应中的 `usage` 结构（输入 token 数）向上传递或暴露给框架层，因而无法统计到具体的 Token 消耗。
- **重排 (Rerank)**：底层重排序 API（如 Cohere 或阿里云百炼的 Rerank）在 AstrBot 适配器中仅解析并返回包含文档索引和相关性得分的 `RerankResult` 列表，同样没有将 Token 消耗信息传递出来，因此也无法记录 Token。
- 某些工具模型
- Provider 没返回 usage 字段

这类数据可以切换到“次数”维度查看。

### 7. 修改配置后没有生效

请重载插件或重启 AstrBot。

## 🛠 维护建议

- 修改插件代码后，建议重载插件或重启 AstrBot。
- 修改统计口径后，建议清空旧数据重新统计，避免新旧口径混在一起。
- 大规模消息环境下，建议谨慎开启高频自动扫描。
- 不建议手动编辑 `usage_stats.db`。
- 如果需要备份统计数据，可以备份整个插件数据目录。
